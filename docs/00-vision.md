# 00 — Vision

## Продукт

Backend для iOS-приложения «Mood Tracker» — психологический ИИ-помощник. Пользователь анонимно (без регистрации) ведёт дневник настроения: выбирает уровень настроения и эмоции, отмечает активности, рассказывает о ситуации текстом или голосом, получает эмпатичный уточняющий вопрос (follow-up) и финальный структурированный анализ с советами от LLM. За завершённые записи начисляются очки и поддерживается streak (серия дней).

## Цели

- Дать iOS-клиенту полный, стабильный, версионированный API для всего жизненного цикла записи настроения.
- Перенести тяжёлую логику (STT, генерация анализа, валидация, начисление очков/streak) на backend, чтобы iOS оставался тонким клиентом.
- Анонимность: идентификация по `device-id` без логина и PII.

## Нефункциональные требования (NFR)

| NFR | Требование |
|---|---|
| Простота | Монолит, минимум инфраструктуры. Команда 1–2 разработчика. |
| Анонимность | Нет регистрации, email, телефона. Скоуп данных строго по `device-id`. |
| Безопасность | Секреты только в env/secret manager. Rate limiting. MIME+signature валидация аудио. Аудио не хранится. |
| Отказоустойчивость LLM | Синхронные вызовы LLM с таймаутом ~30с и 1 retry. Чёткие коды ошибок 502/503 наружу. |
| Консистентность | Начисление очков и streak — атомарно в одной транзакции, идемпотентно. |
| Производительность | История — cursor-пагинация. Индексы под основные запросы. |
| Локализация | Язык ответа LLM управляется клиентом (`language` / `Accept-Language`). |

## Ограничения

- iOS отправляет аудио на backend; STT выполняется на сервере (OpenAI Whisper). Аудио не сохраняется.
- Один follow-up на запись (см. Q-ENTRY-2).
- Follow-up обязателен перед finish (см. Q-ENTRY-1).
- Тексты промтов для LLM зафиксированы дословно (см. ADR-005, `docs/modules/entries`).
