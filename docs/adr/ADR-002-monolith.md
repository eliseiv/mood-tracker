# ADR-002 — Монолитная архитектура

Status: accepted

## Context

Один клиент (iOS), небольшая команда, ограниченный набор доменов (identity, catalog, entries+LLM, gamification). Нет требований к независимому масштабированию частей.

## Decision

Единый FastAPI-монолит с модульной внутренней структурой (`app/{core,db,schemas,api,services,llm,seed}`). Доменное разделение — на уровне модулей/сервисов, не сервисов-процессов.

## Consequences

- (+) Минимум инфраструктуры и операционной сложности, простой деплой и отладка.
- (+) Транзакционная консистентность (finish: analysis + points + streak в одной транзакции).
- (−) При сильном росте отдельные тяжёлые части (LLM) нельзя масштабировать независимо — приемлемо на текущем этапе.

## Alternatives

- Микросервисы (отдельный LLM-сервис) — отклонено как преждевременная сложность для команды 1–2 человека.
