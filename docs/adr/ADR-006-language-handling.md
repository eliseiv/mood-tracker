# ADR-006 — Язык ответа задаётся клиентом

Status: accepted

## Context

LLM должен отвечать (title/overview/advice/follow-up) на языке пользователя. Промт анализа явно требует «Answer in the language I'm asking you about the problem».

## Decision

- iOS передаёт язык явно: поле `language` (BCP-47) в `POST /entries` или заголовок `Accept-Language`.
- Выбранный язык сохраняется в `MoodEntry.language` (на создании записи: `language` тела → `Accept-Language`; иначе `null`) и подставляется в системную часть LLM-запроса для всех шагов записи.
- **Приоритет:** `language` (тело) → `Accept-Language` → серверный автодетект по тексту.
- **Механизм автодетекта (фактическая реализация, `app/llm/language.py`):** `/transcriptions` **stateless** и не привязан к entry, поэтому Whisper-сигнал `detected_language` **не пробрасывается** в запись через сервер. Если на момент LLM-шага язык записи всё ещё `null`, сервер определяет язык **по тексту описания** скрипт-эвристикой Unicode-диапазонов (Cyrillic → `ru`, CJK → `zh`, Кана → `ja`, Hangul → `ko`, Arabic → `ar`, иначе → `en`). Это без дополнительных зависимостей и достаточно для типовых случаев. Результат не сохраняется в `MoodEntry.language` (остаётся резолвом на лету).
- **Как использовать Whisper-detected язык:** чтобы голос-детектированный язык победил, iOS-клиент **передаёт `detected_language` из ответа `/transcriptions` как `language` в `POST /entries`** (ISO 639-1 трактуется как BCP-47 без региона, напр. `ru` → `ru`). Это переводит сигнал Whisper из «серверного автодетекта» в явный клиентский выбор с высшим приоритетом.

## Consequences

- (+) Предсказуемый язык ответа, контролируемый клиентом.
- (+) Один язык на всю запись — консистентность follow-up и анализа.
- (+) Stateless `/transcriptions` не требует хранить состояние языка между запросами.
- (−) Серверный автодетект — грубая скрипт-эвристика (диапазоны Unicode), не различает языки с общей письменностью (en/es/fr/de и т.п. → `en`). Для точности клиент должен передавать `language` (в т.ч. из Whisper).
- (−) Несовпадение языка ввода и заявленного `language` возможно; автодетект — fallback, не переопределяет явный выбор.

## Alternatives

- Пробрасывать `detected_language` через серверное состояние/сессию транскрипции — отклонено: ломает stateless-контракт `/transcriptions` (ADR-004), требует хранения. Вместо этого пробрасывает клиент.
- Тяжёлая ML-библиотека языкового детекта на сервере — отклонено для MVP: лишняя зависимость; клиентский `language` + Whisper покрывают потребность.
- Только автодетект по тексту — отклонено: ненадёжно для коротких сообщений, не уважает выбор пользователя в UI.
