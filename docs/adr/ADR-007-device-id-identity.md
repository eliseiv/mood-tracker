# ADR-007 — Анонимная идентификация по device-id

Status: accepted

## Context

Приложение анонимное, без регистрации и PII. Нужен способ привязать данные к устройству и скоупить доступ.

## Decision

- Заголовок `X-Device-Id` (UUID v4) обязателен на всех endpoint'ах, кроме `/health`.
- Middleware валидирует UUID и делает upsert `Device` (создаёт при первом обращении, обновляет `last_seen_at`).
- Все данные скоупятся по `device-id`. Доступ к чужому ресурсу → `404` (не `403`).
- Логина нет. Переустановка приложения = новый device-id (Q-ID-1); клиент хранит id в Keychain для устойчивости.

## Consequences

- (+) Нулевой friction онбординга, отсутствие PII, простая модель доступа.
- (−) Потеря device-id (без Keychain) = потеря доступа к данным. Документировано для iOS.
- (−) Нет кросс-девайс синхронизации — вне scope на старте.

## Alternatives

- Полноценная авторизация (OAuth/email) — отклонено: противоречит анонимности и усложняет продукт.
- Идентификация по IDFV/IDFA — отклонено: нестабильно, ограничения приватности платформы.
