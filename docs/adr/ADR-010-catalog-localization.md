# ADR-010 — Локализация каталога (EN+RU) явными колонками

Status: accepted

## Context

- Каталог настроений (`MoodScaleLevel`) и эмоций (`Emotion`) нужно отдавать клиенту на двух языках: английском и русском. Ранее `MoodScaleLevel.label` и `Emotion.label` — одна EN-строка.
- Полный каталог обновляется: 5 уровней × 20 эмоций = **100 эмоций**, у каждой EN- и RU-метка (датасет — единственный источник, см. [modules/catalog](../modules/catalog/README.md)).
- Набор языков **фиксирован и мал** (2: `en`, `ru`). Стек кросс-БД: PostgreSQL 16 (prod) + SQLite (local/CI).
- `code` — стабильный машинный ключ, **не локализуется** (клиент шлёт его в `POST /entries`, он же — ключ валидации «эмоция принадлежит уровню»).

## Decision

1. **Хранить локализованные метки явными NOT NULL-колонками** `label_en` и `label_ru` на `Emotion` и `MoodScaleLevel`. Существующая колонка `label` **переименовывается** в `label_en`; добавляется `label_ru`.
2. **`GET /moods` отдаёт одну локализованную строку** в поле `label` по выбранному языку (форма ответа не меняется — поле `label` сохраняется). `code` остаётся стабильным и нелокализованным.
3. **Резолюция языка ответа**: query-параметр `?language=` → заголовок `Accept-Language` → дефолт `en`. Распознаётся **primary-subtag**: `ru`/`ru-RU` → RU-метки; любое другое значение (включая `en`, неизвестные, отсутствие) → EN-метки. Правило применяется единообразно к меткам уровней и эмоций.

## Consequences

- (+) Простота, типобезопасность, тривиальная выборка/сортировка; единый паттерн с уже принятым `Device.locale` → API-поле `language`.
- (+) NOT NULL гарантирует наличие обеих меток у каждой записи каталога.
- (−) Добавление 3-го языка потребует `ALTER TABLE` + миграцию. Для MVP с фиксированными 2 языками — приемлемо; расширение набора языков — отдельный вопрос, не блокирует.

## Alternatives

- **JSONB `labels {en,ru}`** — отвергнут: для фиксированных 2 языков избыточен; усложняет кросс-БД (JSONB в PG vs JSON в SQLite); теряет NOT NULL-гарантию на каждый язык; индексирование/выборка по языку менее тривиальны.
- **Отдельная таблица `translations(entity, entity_id, lang, text)`** — оверинжиниринг для 2 фиксированных языков и небольшого статического каталога.
